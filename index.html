Por que no usar Frameworks o librerias?

Hay muchas razones para decidir usar codigo puro sin un framework en particular. Las principales razones para elegir usar Vanilla JS o PHP simple y plano son:

  1. Performance: Un framework esta hecho para soportar una multitud de casos de uso y contiene muchas librerías que se cargan al inicio de cada petición. El solo hecho de cargarlas para generar una respuesta rápida no hace sentido. Es como salir a comprar a la esquina en automóvil.
  2. Control: El “casarse” con un framework también implica perder una parte del control sobre la implementación.
  3. Detección de errores y fallas: Es mas difícil detectar fallos si estos pueden ser provocados a nivel de framework (externos) y se hace difícil detectarlos y darles seguimiento.
  4. Seguridad: Es posible que un framework tenga fallos de seguridad, y que tarden en ser corregidos. Normalmente uno debería preocuparse de sus propios problemas de seguridad y controlarlos directamente.
  5. Costo de actualización: Cada actualización del framework involucra futuras revisiones en el código y como afectan los cambios externos. Muchas veces, se puede decidir optar por crear un framework propio para tener todo bajo control. A veces un bug o funcionalidades esperadas no pueden esperar un nuevo release.
  6. Elección adecuada: Cada framework apunta a ciertas funcionalidades y puede que no soporten todo lo que deseamos realizar. A veces hasta podría legar a usarse una combinación de ellos. Por ello, algunas compañías prefieren usar sus propias implementaciones ad-hoc.
  7. Especialización Se requiere mayor especialización para usar un framework, La forma de trabajo y la estructura de código es particular de cada uno, esto implica invertir tiempo de desarrollo en el aprendizaje de esa nueva tecnologia para implementarla de manera eficiente en caso de no poseer conocimientos de la misma.
  8. Aprendizaje: Cuando estas aprendiendo o deseas tener mayor control sobre el detalle de la implementación, es mejor trabajar desde el mas bajo nivel posible.
